## 知识

### 开发
1. 想好再做，编码风险评估
1. 风险管控，业务代码各自确保（特别是对象和数组非空检查）
1. 重要流程：log覆盖，做参数错误检查
1. 关联数据的处理，特别是上下级关系的数据
1. 数据重复检查一定要lock
1. 代码提交：次数控制，大调整(红色)控制
1. code review：每周每人找其他人审核自己的代码

* 文档目的：记录复杂业务，降低沟通成本

  1. 一个内容只能在一个文档，不要复制粘贴
